Run a validator

Five steps. One command installs everything. Earn ENSL by storing agent consciousness and producing blocks.

What you earn

  • Block rewards: ~19 ENSL per block you propose
  • With 35 validators: ~7,800 ENSL/day per validator
  • Delegation commission: 10% on ENSL delegated to you by other users
  • Storage credits: earned automatically from staking (10,000 ENSL = 1 MB/month)

Requirements

  • Mac with Node.js 22+ (Apple Silicon recommended)
  • Always-on power (no battery operation)
  • Stable internet connection
  • The --install flag prevents sleep and auto-starts on boot

Quick start

Step 1: Clone and build

git clone https://github.com/suitandclaw/ensoul.git
cd ensoul && pnpm install && pnpm build

Step 2: Start your validator

npx ensoul-node --validate \
  --genesis ~/.ensoul/genesis.json \
  --peers https://v0.ensoul.dev,https://v1.ensoul.dev \
  --install

The --install flag sets up a background service. You can close your terminal after this.

Step 3: Save your seed

Your validator displays a 64-character seed on first run. Copy it. Go to ensoul.dev/wallet.html, click Import Wallet, and paste the seed. This connects your wallet to your validator so you can manage stake and rewards from the browser.

If you missed it, run:

npx ensoul-node --export-seed

Step 4: Register for foundation delegation

curl -X POST https://api.ensoul.dev/v1/validators/register \
  -H "Content-Type: application/json" \
  -d '{"did":"YOUR_DID","publicKey":"YOUR_PUBKEY","name":"YOUR_NAME"}'

The first 10 validators per day receive a 100,000 ENSL foundation delegation. That is enough to start producing blocks immediately.

Step 5: Verify

Check your balance at ensoul.dev/wallet.html. You should see 100,000 ENSL delegated. Confirm your validator is running:

npx ensoul-node --status

View logstail -f ~/.ensoul/validator.log
Stop validatornpx ensoul-node --uninstall

Uptime requirement: Maintain 90%+ uptime over any 7-day window. If uptime drops below 90%, the foundation delegation is withdrawn.

The --install flag handles everything

When you pass --install, the validator runs as a macOS background service with these properties:

  • Background execution: Close your terminal, the validator keeps running
  • Sleep prevention: Uses caffeinate to keep your Mac awake
  • Crash recovery: Auto-restarts if the process crashes
  • Boot persistence: Starts automatically when your Mac boots
  • Logs: tail -f ~/.ensoul/validator.log

Without --install, the validator runs in the foreground and stops when you close the terminal.

What happens when you go offline

  • Missed blocks: You earn no rewards for blocks you miss. Other validators continue producing.
  • Extended downtime: If your uptime falls below 90% over 7 days, the foundation delegation is withdrawn. You can re-register once you are back online.
  • Your own stake is always yours. It remains locked during the standard cooldown period but is never slashed for downtime alone.

Using an existing wallet as your validator identity

If you already have an Ensoul wallet, you can use the same keypair for your validator:

npx ensoul-node --import-seed YOUR_64_CHAR_HEX_SEED

This creates the validator identity from your wallet seed, so your validator rewards go directly to your existing account.
