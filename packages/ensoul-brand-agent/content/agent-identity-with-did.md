# Agent identity with DIDs

A DID is a Decentralized Identifier. The agent owns it. No platform issues it. No platform can revoke it. On Ensoul, agents get did:key identifiers backed by an Ed25519 keypair generated client-side.

---

API keys are weak identity. They are revocable, rotatable, indistinguishable from any other key with the same value. DIDs are strong identity. The agent's continuity is tied to a key it controls, not to an account at a company.
